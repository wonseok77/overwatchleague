## 2. 구현 시 복잡도 포인트 (SPEC.md 기반)

### HIGH 복잡도

1. **팀 밸런싱 알고리즘** (SPEC L233-270)
   - 2단계 알고리즘: 역할군 균형 -> 점수 균형
   - 점수 계산: `(rank_score * 0.4) + (mmr * 0.6)`
   - 반복 스왑 최적화 + 종료 조건 설계 필요
   - 투명성 출력 JSON 구조 포함

2. **시즌 MMR 이월 로직** (SPEC L376-378)
   - 시즌 종료 시 MMR 10% 평균 수렴 계산
   - `이전 1200 -> 1180`, `이전 800 -> 820`
   - 수렴 기준점(1000) 명시 안 됨 -- 열린 질문

3. **멀티 테넌트 데이터 격리** (SPEC L70-78)
   - 모든 쿼리에 `community_id` 필터 필수
   - 크로스 커뮤니티 데이터 유출 방지 필요

### MEDIUM 복잡도

4. **JWT 인증 + Discord OAuth 이중 인증 체계** (SPEC L392-411)
   - 이메일 가입 -> JWT 발급 경로
   - Discord 연동 -> `discord_id` 업데이트 경로
   - 두 경로 간 세션 관리 일관성

5. **파일 업로드 보안** (SPEC L382-388)
   - 확장자 화이트리스트 + MIME 검증
   - 10MB 제한 + 경로 순회 공격 방지
   - 저장 경로: `/uploads/screenshots/{match_id}/`

6. **참가 신청 상태 머신** (SPEC L130-138)
   - `registered -> waitlist -> cancelled -> confirmed` 전이
   - 대기자 자동 승격 로직 (참가자 취소 시)
   - 동시성 제어 (두 명이 동시 신청 시 인원 초과 방지)

## 3. 팀 밸런싱 알고리즘 엣지 케이스 목록

향후 테스트 시 반드시 검증해야 할 시나리오:

### 인원 관련
- [ ] 참가자가 정확히 10명 (5v5 정상 케이스)
- [ ] 참가자가 10명 미만 (팀 구성 불가? 불균등 허용?)
- [ ] 참가자가 홀수 (예: 11명 -- 한 팀이 6명?)
- [ ] 참가자가 12명 이상 (대기자 처리 후 10명만?)

### 역할군 관련
- [ ] 탱커 0명 -- 역할군 균형 불가 시 처리
- [ ] 서포트 1명 -- 양 팀에 배분 불가
- [ ] 전원 DPS -- 역할군 균형 단계 스킵?
- [ ] 역할군 미입력 참가자 혼재
- [ ] 한 역할군에 3명 이상 편중 (DPS 7명)

### 점수 관련
- [ ] 전원 동일 MMR (완전 균형 -- 역할군만 기준)
- [ ] MMR 극단적 편차 (한 명만 2000, 나머지 800)
- [ ] rank_score 미입력 (current_rank NULL) -- 점수 계산 폴백
- [ ] 신규 유저 (MMR 기본값 1000) 다수 참여

### 알고리즘 수렴
- [ ] 스왑 반복이 무한루프에 빠지는 케이스
- [ ] 임계값 미달 상태에서 반복 횟수 초과
- [ ] 최적해가 여러 개인 경우 결정론적 결과 보장

### MMR 변동
- [ ] 승리 시 상대팀 점수가 극단적으로 높을 때 보너스 상한
- [ ] 패배 시 상대팀 점수가 극단적으로 낮을 때 페널티 하한
- [ ] 무승부 처리 (MMR 변동 0 확인)
- [ ] MMR이 0 이하로 내려가는 케이스 (하한선 필요?)

## 4. 인증 보안 체크포인트

구현 완료 후 반드시 검증할 항목:

### JWT
- [ ] HS256 서명 검증 (SECRET_KEY 노출 방지)
- [ ] 토큰 만료 7일 정확히 적용
- [ ] Refresh Token 구현 여부 (SPEC에 언급되나 상세 없음)
- [ ] 만료된 토큰으로 API 호출 시 401 반환

### 비밀번호
- [ ] bcrypt cost factor 12 적용
- [ ] 비밀번호 최소 길이/복잡도 정책 (SPEC 미명시)
- [ ] 비밀번호 평문 로깅 방지

### 권한
- [ ] admin 전용 엔드포인트 (경기 결과 입력, 팀 구성 등) member 접근 차단
- [ ] 다른 커뮤니티의 admin이 타 커뮤니티 자원 접근 불가
- [ ] community_id 기반 데이터 격리 모든 API에 적용

### 파일 업로드
- [ ] JPG/PNG/WebP만 허용 (확장자 + MIME)
- [ ] 10MB 초과 파일 거부
- [ ] 경로 순회 공격 (`../` 포함 파일명) 방지
- [ ] 업로드된 파일 실행 권한 제거

### CORS
- [ ] 프론트엔드 도메인만 허용 (와일드카드 * 사용 금지)
- [ ] credentials 포함 요청 처리

### Discord OAuth
- [ ] state 파라미터로 CSRF 방지
- [ ] 콜백 URL 검증
- [ ] discord_id 업데이트 시 기존 연동 해제 처리
