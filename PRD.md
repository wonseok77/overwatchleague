# PRD: 오버워치 커뮤니티 내전 종합 플랫폼

> 작성일: 2026-03-05 | 최종 수정: 2026-03-06
> 상태: v2.0 (Phase 5 완료)

---

## 1. 한 줄 포지셔닝

> **"오버워치 내전을 취소 없이 지속시키는 커뮤니티 활성화 엔진 — 팀 밸런싱부터 스탯 아카이브까지"**

---

## 2. 문제 정의

### 핵심 문제
오버워치 커뮤니티 내전은 운영자 번아웃과 참여 동기 부족으로 흐지부지되는 패턴이 반복된다.

- **운영자**: 매주 구글폼 → 수동 팀 구성 → 결과 기록의 반복 노동. 티어 신뢰도 낮음 + 역할군 분포 무시로 밸런스 불만 발생.
- **참가자**: 스탯/기록이 쌓이지 않아 내전 참여가 의미 없게 느껴짐. 팀 배정 과정이 불투명해 공정성 불신.
- **커뮤니티 전체**: 기록 없는 경기는 소비되고 잊혀진다. 성장과 경쟁의 피드백 루프가 없어 리텐션이 낮다.

### 진짜 원인 (인터뷰 발견)
- 참여율이 높았던 순간은 "공휴일 + 특별 외부 스크림"이었다 → **이벤트성이 단기 참여율에 영향**
- 그러나 지속적 참여를 만드는 것은 **스탯/레코드 누적** (리텐션 핵심, 사용자 직접 확인)
- 운영자가 매주 팀을 수동으로 짜는 공수가 지속 불가능한 수준 → **자동화 없이는 운영 지속 불가**

---

## 3. 킬러 인사이트

### Insight 1: 운영 마찰 제거가 활성화의 선결 조건
스탯이 없어서가 아니라, 내전을 여는 것 자체가 힘들어서 취소된다. 참가 신청 자동화 + 팀 자동 밸런싱이 없으면 스탯을 아무리 잘 만들어도 내전 자체가 열리지 않는다.

### Insight 2: 공정성이 아니라 투명성이 핵심
팀 밸런싱 불만의 근원은 실제 불공정보다 과정의 불투명함이다. 알고리즘이 어떤 근거로 팀을 구성했는지 참가자에게 보여주는 것만으로 수용도가 크게 올라간다.

### Insight 3: 스탯 = 개인 성장 증거 + 경쟁심 연료
스탯이 쌓이면 "내가 얼마나 성장했나" + "다른 멤버와 비교" 두 가지 동기가 동시에 생긴다. 이것이 다음 내전에 참여하게 만드는 리텐션 루프다. 게임 내 랭크 시스템의 커뮤니티 버전.

### Insight 4: Blizzard API 공백 = 차별화 기회
기존 모든 경쟁 서비스(Overbuff, MasterOverwatch 등)가 공식 API에 의존하다 아카이브됐다. 스크린샷 업로드 기반 하이브리드 방식은 API 의존도를 없애 서비스의 지속 가능성을 확보한다.

### Insight 5: 운영자 번아웃 방지 = 커뮤니티 생존
운영자가 매주 팀 짜는 공수를 자동화로 줄여줄 때 리그가 지속된다. 운영자의 지속 가능성이 커뮤니티 생존의 유일한 조건이다.

### Insight 6: 시즌제 운영
시즌마다 스탯과 파워랭킹을 포지션별로 집계해서 다음시즌에 대한 퍼포먼스 향상심을 고취한다.

## 4. 가정 위험도 매트릭스

| 가정 | 위험도 | 검증 방법 |
|------|--------|-----------|
| 스탯 시스템이 리텐션을 높인다 | 높음 (미검증) | MVP 출시 후 3개월 리텐션 측정 |
| 팀 자동 밸런싱이 불만을 줄인다 | 중간 | 배정 결과 투명성 공개 후 불만 횟수 비교 |
| 스크린샷 업로드 방식이 참가자 부담이 되지 않는다 | 중간 | 업로드율 추적 (50% 미만이면 다른 방식 고려) |
| 다른 커뮤니티가 자연스럽게 유입된다 | 높음 (PLG 전략 미검증) | 6개월 내 다른 커뮤니티 2개 이상 자가 가입 여부 |
| 현시즌 티어보다 내전 누적 스탯 기반 밸런싱이 더 신뢰받는다 | 낮음 | 배정 불만 비율로 검증 |

## 6. 사용자 프로필

### Primary: 내전 운영자 (동시에 참가자)
- 오버워치 커뮤니티(Discord/카카오)를 직접 운영
- 매주 팀 구성, 참가 신청 관리, 결과 기록의 반복 노동 경험
- 가장 원하는 것: 팀 짜는 공수 제거 + 커뮤니티 활성화

### Secondary: 내전 참가자
- 정기적으로 내전에 참여하는 멤버
- 가장 원하는 것: 내 스탯/성장이 기록되고 인정받는 것 + 공정한 팀 배정

## 7. 핵심 기능 요구사항

### P0: MVP — 팀 자동 밸런싱 ✅ 완료
- 참가자 등록 시 역할군(Tank/DPS/Support) + 내전 누적 승률/ELO 입력
- 자동 팀 구성 알고리즘: 역할군 균형 + 실력 균형 동시 고려
- 배정 결과에 "왜 이 팀 구성인가" 근거 표시 (투명성)
- 수동 조정 옵션 (운영자 override)

### P0: 참가 신청 자동화 ✅ 완료
- 내전 일정 등록 → 참가 신청 링크 생성 → Discord 웹훅 알림 연동
- 참가/취소/재신청 플로우 (구글폼 대체)
- 마감 시 자동 팀 구성 트리거
- 대기자 자동 관리 (인원 초과 시 대기 등록)

### P1: 경기 결과 + 스탯 기록 ✅ 완료
- 경기 결과 (승/패, 맵, 날짜) 운영자 입력
- 스코어카드 스크린샷 업로드 → OCR 자동 추출 (Tesseract)
- 각 멤버별 플레이한 영웅 Most 3 기록
- 경기 목록 아카이브 (시즌/날짜 필터)

### P1: 개인 프로필 + 커뮤니티 리더보드 ✅ 완료
- 멤버 프로필: 내전 누적 승/패, 승률, 자주 픽한 영웅 Top 3, 시즌별 기록
- 커뮤니티 파워랭킹 (내전 ELO 기반)
- 시즌 리셋 + 시즌 기록 아카이브

### P1+: 매치메이킹 고도화 ✅ 완료 (Phase 5)
- **세션 기반 내전 관리**: 하루 단위 세션 생성, 세션 내 복수 게임 자동 편성
- **3단계 매치메이킹 알고리즘**: 참여 횟수 균등 분배 → 포지션 배정(1/2/3지망) → 4가중치 팀 밸런싱
- **4가중치 밸런싱**: 포지션별 티어(0.3) + MMR(0.4) + 승률(0.2) + 역할 스탯(0.1)
- **포지션별 랭크**: 탱커/딜러/서포터 각각 독립 티어 설정 (Bronze~Champion)
- **매치메이킹 미리보기**: 게임별 팀 구성 + 밸런스 점수 + 대기자 사유 시각화
- **OCR 스탯 추출**: Tesseract로 스코어보드 스크린샷 자동 인식 (K/D/A/DMG/HEAL)
- **Discord 알림**: 매치메이킹 확정 시 팀 구성 자동 전송

### P2: 하이라이트 클립 아카이브 ✅ 완료
- 경기당 클립 URL (YouTube/트위치) 등록
- 플레이어별 하이라이트 모아보기
- 커뮤니티 명장면 피드

### P2: 멀티 커뮤니티 (멀티 테넌트) — 미구현
- 커뮤니티 생성 (워크스페이스 개념)
- 커뮤니티별 독립 멤버/스탯/설정
- 공개 커뮤니티 디렉토리 (발견 가능성)

## 8. 비기능 요구사항

- **Blizzard API 독립**: 외부 API 의존 없이 동작하는 구조 필수
- **모바일 친화적 UI**: 내전 당일 스마트폰으로 참가 신청/결과 확인
- **Discord 연동**: Discord OAuth 로그인 + 웹훅 알림 (커뮤니티가 이미 Discord에 있음)
- **데이터 내구성**: 운영자 탈퇴/봇 중단에도 데이터 유실 없음 (기존 Discord 봇의 핵심 약점)

## 11. 리스크

1. **스탯 입력 피로**: 스크린샷 업로드가 귀찮으면 스탯 데이터가 쌓이지 않는다 → 최소 입력 (승/패 + 영웅 3개)로 진입 장벽 낮춤
2. **팀 밸런싱 불만 지속**: 알고리즘 결과에도 불만이 있을 수 있다 → 투명한 이유 표시 + 이의 제기 플로우 설계
3. **PLG 성장 불확실**: "알아서 가입"은 검증되지 않은 가설 → 파일럿 3개월 후 평가 후 홍보 전략 추가 고려
4. **운영자 단일 의존**: 운영자 1명이 떠나면 커뮤니티가 무너질 수 있다 → 공동 운영자 권한 기능 필수
